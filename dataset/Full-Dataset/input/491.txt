The system shall require explicit permission controls among different users, user groups, and roles to ensure security and compliance. The Access Control System (ACS) shall assist organizations in managing permissions across various levels, ensuring each user has appropriate access within the system, thereby preventing sensitive information leakage and misuse of operations.

The core objectives of the system include managing user information, roles, permissions, and policies. It shall provide system administrators with convenient user permission configuration tools while supporting the dynamic adjustment of permission relationships between users and roles. Common application scenarios include Content Management Systems (CMS), internal management platforms, and Customer Relationship Management (CRM) systems.

Primary functional requirements:
User management:
Registration, management, updating, and deletion of users.
Each user shall possess a unique identifier (such as username or email) and personal information.
Role management:
The system shall support the creation, management, and deletion of roles.
Each role shall correspond to a set of permissions, which can be assigned to a role by an administrator, and then assigned to users.
Permission management:
Permissions may be set for operations, such as 'view', 'edit', 'delete', and 'create'.
Permissions shall include access control to certain functional modules or data resources within the system.
