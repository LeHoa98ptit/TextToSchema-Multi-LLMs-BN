The database is a software project management system primarily designed to manage the development process of software projects, task assignments for team members, code submissions, and project version management. The key tables involved in the system include: User Table, Project Table, Project Version Table, Task Table, Code Submission Table. Below is a brief description of the database function.

1.User Management:   
Records the basic information of each user in the system, who may be a project manager, developer, or tester.  
Fields: User ID, User Name, Password, Email, Phone, Role, Registration Time, Last Login Time

2.Project Management: 
Records the basic information of software development projects, each project has a responsible person and manages its current status.  
Fields: Project ID, Project Name, Project Introduction, Responsible Person ID, Creation Time, Project Status

3. Project Version Management:
Records the version information of each project, including version number, release date, etc. Each version is associated with a specific project.  
Fields: Version ID, Project ID, Version Name, Release Date, Version Description, Is Stable Version

4. Task Management:
Records the assignment and progress of tasks within the project; tasks are the work that developers need to complete. Each task has a start time, end time, and task status.  
Fields: Project ID, Assigned To User ID, Task Name, Task Status, Start Time, End Time, Task Description

5. Code Submission Management:  
Records the code information submitted by developers to the project, including submission time, submission description, code path, etc.  
Fields: User ID, Project ID, Submission Time, Submission Description, Code Path
