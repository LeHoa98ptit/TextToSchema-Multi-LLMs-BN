User Authentication and Management System
This system is designed to manage and authenticate users while tracking their interactions with various services and worker nodes in the system. The key functionalities of the system are:
1.User Registration and Management:
Store user registration information, including username, full name, email, password hash, registration date, and last login date.
Ensure the uniqueness of usernames and email addresses.
2.User Authentication:
Store information of authenticated users, including username, full name, authentication level (ring), and authentication time.
Ensure the uniqueness of authenticated usernames.
Record signatures and issuer information during the authentication process.
3.User Migration:
Track user migrations from an old system to the new system, recording the old and new user IDs, along with the migration time.
4.Authenticated Consumer Management:
Store information of authenticated consumers, including name, description, type, and associated user ID.
Store configuration data,permissions, and signatures for each consumer.
5.Session Management:
Store session information, including session ID, associated consumer ID, expiration time, and MFA (Multi-Factor Authentication) status.
Record session signatures and issuer information.
6.Service Management:
Store information of various services in the system, including service name, type, URL, and configuration.
Track the last heartbeat time and monitoring status of services.
Record service signatures and issuer information.
7.Worker Node Management:
Store information of worker nodes, including node name, status, associated service ID, operating system, architecture, and last heartbeat time.
Record worker node signatures and issuer information.
Each service contains multiple worker nodes, each user may have multiple authenticated consumers, and each consumer will have multiple sessions.
